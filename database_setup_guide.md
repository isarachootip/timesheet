# คู่มือการตั้งค่าสิทธิ์เข้าถึงฐานข้อมูล PostgreSQL (Hostinger VPS)

เนื่องจากสิทธิ์ความปลอดภัยเริ่มต้นของ PostgreSQL จะอนุญาตเฉพาะการเข้าใช้งานภายในเครื่อง (Localhost) เท่านั้น เอกสารฉบับนี้สรุปวิธีการเปิดสิทธิ์เชื่อมต่อระยะไกลเพื่อให้ Coolify สามารถคุยกับฐานข้อมูลบน VPS `187.77.147.16` ได้ครับ

---

## ขั้นตอนที่ 1: การอนุญาตสิทธิ์เข้าถึงระยะไกล (Remote Access) ผ่าน SSH

กรุณาใช้โปรแกรม Terminal (เช่น PowerShell, Command Prompt, หรือ PuTTY) เพื่อเชื่อมต่อ SSH ไปยังเซิร์ฟเวอร์ Hostinger ของคุณ:

```bash
ssh root@187.77.147.16
```

เมื่อเชื่อมต่อสำเร็จแล้ว ให้ดำเนินการแก้ไขไฟล์ตามขั้นตอนด้านล่างนี้:

### 1.1 แก้ไขไฟล์ตั้งค่าเครือข่าย (`postgresql.conf`)
เปิดไฟล์ตั้งค่าหลักด้วยโปรแกรมเขียนข้อความ `nano` (ระบบอาจใช้เวอร์ชันอื่น เช่น 14, 15 หรือ 16 ให้เปลี่ยนเลขโฟลเดอร์ตามเวอร์ชันที่ใช้จริง):

```bash
sudo nano /etc/postgresql/15/main/postgresql.conf
```

- มองหาบรรทัด: `#listen_addresses = 'localhost'`
- แก้ไขเป็น: `listen_addresses = '*'` (ลบเครื่องหมาย `#` ด้านหน้าออก)
- *คำอธิบาย: เพื่อสั่งให้ PostgreSQL รับการเชื่อมต่อจากทุก IP*
- กด `Ctrl + O` แล้วตามด้วย `Enter` เพื่อบันทึกไฟล์ จากนั้นกด `Ctrl + X` เพื่อออก

### 1.2 แก้ไขไฟล์ตั้งค่าความปลอดภัย (`pg_hba.conf`)
เปิดไฟล์สิทธิ์ผู้ใช้งาน:

```bash
sudo nano /etc/postgresql/15/main/pg_hba.conf
```

- เลื่อนลงไปด้านล่างสุดของไฟล์ แล้วเพิ่มบรรทัดนี้ต่อท้าย:
  ```text
  # เปิดสิทธิ์การเข้าถึงจากทุก IP สำหรับแอปพลิเคชัน Coolify
  host    timesheet_db    isara_admin     0.0.0.0/0               md5
  ```
- *คำอธิบาย: อนุญาตให้ผู้ใช้ `isara_admin` เข้าใช้งานฐานข้อมูล `timesheet_db` จากไอพีภายนอกใดก็ได้*
- กด `Ctrl + O` และกด `Enter` เพื่อบันทึก จากนั้นกด `Ctrl + X` เพื่อออก

### 1.3 รีสตาร์ทการทำงานของ PostgreSQL
รันคำสั่งนี้เพื่อให้ค่าคอนฟิกใหม่เริ่มทำงาน:

```bash
sudo systemctl restart postgresql
```

## ขั้นตอนที่ 2: การเปิดพอร์ตผ่านระบบไฟร์วอลล์ (UFW Firewall)

หาก VPS ของคุณเปิดใช้งานระบบ Firewall พอร์ตเชื่อมต่อ `5432` ของ PostgreSQL อาจถูกบล็อกอยู่ ให้เปิดพอร์ตด้วยคำสั่งนี้:

```bash
# ตรวจสอบสถานะไฟร์วอลล์
sudo ufw status

# เปิดพอร์ต PostgreSQL
sudo ufw allow 5432/tcp

# โหลดไฟร์วอลล์ใหม่
sudo ufw reload
```

## ขั้นตอนที่ 3: ทดสอบการเชื่อมต่อหลังจากแก้ไขเสร็จสิ้น

คุณสามารถกลับมาที่เครื่องคอมพิวเตอร์ของคุณ แล้วรันคำสั่งทดสอบเชื่อมต่อที่ผมเขียนเตรียมไว้ให้ได้ทันที ผ่าน PowerShell:

```bash
node scratch/test_db.js
```

หากการตั้งค่าถูกต้อง หน้าจอจะแสดงข้อความสีเขียว:
`Successfully connected to the database!`
หลังจากนั้นระบบเว็บในมือถือและคอมพิวเตอร์ของคุณจะทำการดึงข้อมูลชุดเดียวกันและทำงานร่วมกันได้ทันทีครับ!
